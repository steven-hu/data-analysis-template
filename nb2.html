As you can see, in the entire database there are 889 non-cancelled flights with departure delays of less than -60 minutes, and 17099 flights with non-cancelled departure delays of over 500 minutes. Therefore, based on the graphical representation and these figures, it seems reasonable to use these cutoffs to remove extreme outliers.

In [25]: 
import csv

airports = [u'ORD', u'ATL', u'DFW', u'LAX', u'PHX', u'DEN', u'DTW', u'IAH', u'MSP', u'SFO', u'STL', u'EWR', u'LAS', u'CLT', u'LGA', u'BOS', u'PHL', u'PIT', u'SLC', u'SEA']
for i in airports:
    read = open('../data/raw/%s.csv' % i,'r')
    write = open('../data/cleaned/%s.csv' % i, 'w')
    reader = csv.reader(read)
    writer = csv.writer(write)
    for j in reader:
        if j[3] == 'NA':
            continue
        elif int(j[3]) < 0:
            if int(j[3]) < -60:
                continue
            else:
                j[3] = 0
                writer.writerow(j[:4])
        elif int(j[3]) > 500:
            continue
        else:
            writer.writerow(j[:4])
    read.close()
    write.close()

This writes new, cleaned csv files. Cancelled flights are removed, flights outside (-60, 500) for delay are removed, and all negative delays (early flights) are set to 0. This is because in evaluating delayed flights, an early flight is essentially the same as an on-time flight. It doesn't make sense to say that an airport that has 3 flights, with delays -5, 0, and 5, has an average delay of 0 minutes.

Other than this stuff, the data we need is all in good shape. No issues like missing fields.
